### Purpose & Functionality
* **The Goal:** To provide a lightweight and efficient way for users to book resources or appointments through a web interface.
* **Key Solution:** Simplifies the scheduling process by replacing manual booking with an automated system that prevents double-booking and tracks reservation history.

---

### Core Features
* **Interactive Booking Calendar:** View and select available dates/time-slots.
* **User Authentication:** Secure login for users to manage their own reservations.
* **Admin Dashboard:** Centralized view for administrators to approve, cancel, or modify bookings.
* **Database Driven:** Real-time availability updates powered by a MySQL backend.

### Installation & Setup
1. **Clone Repository:** `git clone https://github.com/ratchanon-noknoy2318/Booking-Project.git`
2. **Database Setup:** Import the `.sql` file found in the project directory into your MySQL server.
3. **Configuration:** Set your database credentials in the configuration file (e.g., `dbconnect.php`).
4. **Run:** Access the project via a local server (XAMPP/Apache).
